Safhat Template Framework

Reusable Field Documentation for Building Templates

This page is not a wedding, graduation, or company template. It is a developer reference page that shows every available field type, how to use it in HTML, how to define it in template.json, and what properties it supports.

01

Fast Template Workflow

Use the starter kit in this folder to create a clean, platform-ready template without rebuilding the same boilerplate each time.

1

Scaffold

Generate a template folder with linked HTML, CSS, and template.json.

node scaffold-template.mjs my-template --name "My Template" --direction rtl
2

Design + bind

Build the layout, then replace editable content with matching {{fieldKey}} variables.

<h1>{{title}}</h1>
3

Validate

Catch duplicate keys, missing files, unsupported fields, and unmatched variables before publishing.

node validate-template.mjs ../my-template
One rule that prevents most template bugs: every editable {{variable}} in HTML must have exactly one field with the same root key in template.json.

04

Basic Field Structure

Every editable value in the HTML should have a matching field in template.json.

HTML usage

Use Mustache-style variables in your HTML.

HTML 
<h1 data-section="title">{{title}}</h1>

<p data-section="description">
    {{description}}
</p>

JSON usage

Define the same keys inside the fields array.

template.json 
{
  "fields": [
    {
      "key": "title",
      "label": "Title",
      "type": "text",
      "required": true,
      "defaultValue": "My Title"
    },
    {
      "key": "description",
      "label": "Description",
      "type": "textarea",
      "required": true,
      "defaultValue": "Short description here"
    }
  ]
}
05

Common Field Properties

These properties can be used with most field types.

Property Type Purpose
key string The variable name used in HTML like {{title}}.
label string The field label shown to the user in the form.
type string The field type: text, image, select, color, repeater, etc.
required boolean Makes the field required.
defaultValue any The default value used before the user edits the field.
placeholder string Text shown inside empty inputs.
hint string Small help text under the field.
formTab string Groups fields into tabs: content, design, media, features, settings.
group string Groups related fields inside the same tab.
order number Controls field order in the form.
width string Form width: full, half, third.
visibleWhen object Shows the field only when a condition is true.
requiredWhen object Makes the field required only when a condition is true.
validationMessages object Custom error messages for validation.
Common field example 
{
  "key": "title",
  "label": "Title",
  "type": "text",
  "formTab": "content",
  "group": "main",
  "order": 1,
  "width": "full",
  "required": true,
  "defaultValue": "Example Title",
  "placeholder": "Write title here",
  "hint": "This text appears as the main title",
  "minLength": 3,
  "maxLength": 80,
  "trim": true,
  "sanitize": true,
  "validationMessages": {
    "required": "Title is required",
    "minLength": "Title is too short",
    "maxLength": "Title is too long"
  }
}
06

Validation Properties

Use these properties to control data quality before rendering the template.

Text validation

Use for names, titles, short descriptions.

minLength / maxLength / pattern

Number validation

Use for age, price, count, percentage.

min / max / step

List validation

Use for skills, tags, services.

minItems / maxItems / uniqueItems

File validation

Use for images and uploaded media.

accept / maxSizeMb / aspectRatio
Validation example 
{
  "key": "username",
  "label": "Username",
  "type": "text",
  "required": true,
  "minLength": 3,
  "maxLength": 30,
  "pattern": "^[a-zA-Z0-9_]+$",
  "patternMessage": "Only letters, numbers, and underscore are allowed",
  "trim": true,
  "sanitize": true,
  "validationMessages": {
    "required": "Username is required",
    "minLength": "Username must be at least 3 characters",
    "maxLength": "Username must not exceed 30 characters",
    "pattern": "Username format is invalid"
  }
}
07

Conditional Visibility and Required Rules

Use conditions when a field should appear only after another field value is selected.

visibleWhen

Use it when the field should be hidden or shown based on another field.

visibleWhen 
{
  "key": "heroImage",
  "label": "Hero Image",
  "type": "image",
  "visibleWhen": {
    "field": "showHeroImage",
    "equals": "yes"
  }
}

requiredWhen

Use it when the field becomes required only in a specific case.

requiredWhen 
{
  "key": "heroImage",
  "label": "Hero Image",
  "type": "image",
  "requiredWhen": {
    "field": "showHeroImage",
    "equals": "yes"
  }
}

image

Image Field

Use for hero images, profile pictures, logos, product images, event images, backgrounds, gallery items, certificates, or any uploaded visual.

How image works

In HTML, the image field value is used inside the src attribute. In template.json, you can control upload type, file size, aspect ratio, crop, width, and conditional visibility.

HTML 
<section class="hero-media" data-section="heroImage">
    <img src="{{heroImage}}" alt="{{heroImageAlt}}" />
</section>
template.json 
{
  "key": "heroImage",
  "label": "Hero Image",
  "type": "image",
  "formTab": "media",
  "group": "hero",
  "required": true,
  "defaultValue": "https://placehold.co/900x600/eef2ff/2563eb?text=Hero+Image",
  "placeholder": "Upload hero image",
  "hint": "Recommended size: 900x600 or larger",
  "accept": [
    "image/png",
    "image/jpeg",
    "image/webp"
  ],
  "maxSizeMb": 5,
  "minWidth": 600,
  "minHeight": 400,
  "aspectRatio": "3:2",
  "crop": true,
  "validationMessages": {
    "required": "Image is required",
    "accept": "Only PNG, JPG, and WEBP are allowed",
    "maxSizeMb": "Image must be less than 5MB",
    "minWidth": "Image width is too small",
    "minHeight": "Image height is too small"
  }
}
Image Property Example Purpose
accept ["image/png", "image/jpeg"] Controls allowed file types.
maxSizeMb 5 Maximum upload size.
minWidth 600 Minimum image width.
minHeight 400 Minimum image height.
aspectRatio 1:1, 3:4, 16:9, auto Recommended or required crop ratio.
crop true Allows image cropping before saving.
Supports: accept maxSizeMb minWidth minHeight aspectRatio crop visibleWhen requiredWhen
url

URL Field

Use for website links, portfolio links, social links, project links, external buttons, or contact links.

HTML

</a>

  "pattern": "^(#|https?:\\/\\/.+)$",
  "patternMessage": "URL must be # or start with http or https"
}

Feature Blocks

Use feature blocks for built-in interactions like messages, feelings, RSVP, votes, or forms.

HTML feature block 
<section data-section="messages" data-feature="feelings">

    <input type="text" placeholder="Your name" data-feeling-name />

    <textarea placeholder="Write message" data-feeling-message></textarea>

    <button type="button" data-send-feeling>
        Send
    </button>

    <p data-feeling-result></p>

    <div data-feelings-list></div>

</section>
template.json features 
{
  "features": {
    "feelings": {
      "enabled": true,
      "approvalMode": "manual",
      "preventDuplicate": true,
      "rateLimitMinutes": 5,
      "maxNameLength": 60,
      "maxMessageLength": 220,
      "title": "Messages",
      "description": "Visitors can write messages"
    },
    "rsvp": {
      "enabled": true,
      "preventDuplicate": true,
      "maxNameLength": 60,
      "title": "Visitor Response",
      "description": "Visitors can choose a response"
    }
  }
}
09

Sections

Sections are used by your builder to select, hide, move, resize, and style parts of the template.

HTML section 
<section class="hero" data-section="hero">
    <h1 data-section="heroTitle">{{heroTitle}}</h1>
    <p data-section="heroDescription">{{heroDescription}}</p>
</section>
template.json section 
{
  "key": "heroTitle",
  "label": "Hero Title",
  "selector": "[data-section='heroTitle']",
  "type": "text",
  "editable": true,
  "visible": true,
  "movable": true,
  "resizable": false,
  "styleControls": [
    "fontSize",
    "fontWeight",
    "color",
    "textAlign",
    "lineHeight",
    "margin"
  ]
}
